# Flight-App
For class "Internet Programming".


1. Αρχιτεκτονική της Διαδικτυακής Εφαρμογής
Η παρούσα διαδικτυακή εφαρμογή σχεδιάστηκε και αναπτύχθηκε με βάση την
κλασική Τριεπίπεδη Αρχιτεκτονική (3-Tier Architecture), η οποία εξασφαλίζει την
πλήρη αποσύνδεση της διεπαφής χρήστη, της επιχειρηματικής λογικής και της
διαχείρισης των δεδομένων. Η δομή αυτή επιτρέπει την εύκολη συντήρηση, την
ασφάλεια και την επεκτασιμότητα του συστήματος.

Α. Επίπεδο Παρουσίασης (Presentation Tier / Frontend)
Αποτελεί το ανώτερο επίπεδο της εφαρμογής, υπεύθυνο για την άμεση αλληλεπίδραση
με τον τελικό χρήστη (Client-Side Interface). Για την υλοποίησή του υιοθετήθηκαν
σύγχρονες τεχνολογίες ιστού με στόχο τη βέλτιστη εμπειρία χρήστη (UX) και τη
διευκόλυνση της πλοήγησης:
● HTML5 & CSS3: Χρησιμοποιήθηκαν για τη θεμελιώδη δομή, τη μορφοποίηση
και τη σημασιολογική οργάνωση του περιεχομένου των ιστοσελίδων.
● Framework Bootstrap 5: Ενσωματώθηκε για τη σχεδίαση μιας σύγχρονης,
καλαίσθητης και πλήρως αποκρινόμενης διεπαφής. Η χρήση του διασφαλίζει ότι η
εφαρμογή προσαρμόζεται αυτόματα και λειτουργεί υποδειγματικά σε οποιοδήποτε
μέγεθος οθόνης (σταθεροί υπολογιστές, tablet, έξυπνα κινητά). Έγινε επίσης
αρκετή χρήση των bootstrap icons.
● JavaScript: Αξιοποιήθηκε για την εκτέλεση άμεσων ελέγχων εγκυρότητας
δεδομένων (Validation) στις φόρμες πριν από την υποβολή τους, καθώς και για τη
διαχείριση διαδραστικών στοιχείων. Επιπλέον, προσφέρει ένα επιπλέον επίπεδο
ασφάλειας μέσω προειδοποιητικών παραθύρων επιβεβαίωσης σε κρίσιμες
ενέργειες (όπως η οριστική ακύρωση κράτησης ή η διαγραφή στοιχείων),
αποτρέποντας λάθη από την πλευρά του χρήστη.

Β. Επίπεδο Λογικής Εφαρμογής (Backend)
Αποτελεί τον κεντρικό πυρήνα του συστήματος στον εξυπηρετητή (Server-Side). Στο
επίπεδο αυτό υλοποιήθηκε το σύνολο της επιχειρηματικής λογικής της εφαρμογής με τη
χρήση της γλώσσας προγραμματισμού PHP (PHP Interpreter). Το Backend αναλαμβάνει
να γεφυρώσει την οπτική διεπαφή (Frontend) με τη βάση δεδομένων, επεξεργαζόμενο τα
αιτήματα HTTP και επιβάλλοντας τους απαραίτητους περιορισμούς ασφαλείας και
λειτουργίας.
Οι βασικοί άξονες ανάπτυξης του επιπέδου αυτού περιλαμβάνουν:
● Διαχείριση Συνεδρίων (Session Management) & Αυθεντικοποίηση:
Χρησιμοποιήθηκε ο μηχανισμός session_start() της PHP για την
παρακολούθηση της κατάστασης των χρηστών. Κατά την επιτυχή σύνδεση
(Login), τα στοιχεία διαπιστευτηρίων επαληθεύονται, κρυπτογραφούνται και
αποθηκεύονται με ασφάλεια στο Session. Παράλληλα, υλοποιήθηκε έλεγχος
δικαιωμάτων βάσει ρόλων (Role-Based Access Control - RBAC) στην κορυφή
κάθε αρχείου, αποτρέποντας την απευθείας πρόσβαση σε σελίδες διαχείρισης
μέσω URL (Privilege Escalation).
● Επιβολή Επιχειρηματικών Κανόνων (Business Logic): Ο κώδικας PHP ελέγχει
δυναμικά τις καταστάσεις των οντοτήτων. Για παράδειγμα, εμποδίζεται η
ακύρωση εισιτηρίων ή η αλλαγή στοιχείων όταν μια πτήση έχει ήδη
«ΠΕΡΑΤΩΘΕΙ» ή «ΑΚΥΡΩΘΕΙ». Αντίστοιχα, επιβάλλεται ο περιορισμός
επιλογής θέσεων, επιτρέποντας την αποδέσμευση ή αλλαγή καθίσματος
αποκλειστικά σε κατόχους Business και Normal εισιτηρίων, αποκλείοντας τους
επιβάτες τύπου Economy.
● Υλοποίηση Αλυσιδωτών Μεταβολών (Database Transactions): Για τη
διασφάλιση της ακεραιότητας και της συνέπειας των δεδομένων,
χρησιμοποιήθηκαν Transactions της PHP ($pdo->beginTransaction(),
commit(), rollBack()). Όταν ένας διαχειριστής ακυρώνει μια πτήση, το
Backend αναλαμβάνει να ακυρώσει αυτόματα όλες τις σχετικές κρατήσεις των
επιβατών και να απελευθερώσει τις θέσεις τους. Σε περίπτωση αποτυχίας
οποιουδήποτε ενδιάμεσου ερωτήματος, εκτελείται αυτόματη αναίρεση (Rollback)
των αλλαγών.
● Αυτοματοποιημένη Μαζική Εισαγωγή Δεδομένων (CSV Parsing):
Αναπτύχθηκε μηχανισμός επεξεργασίας αρχείων μέσω της συνάρτησης
fgetcsv(), επιτρέποντας στους διαχειριστές να εισάγουν μαζικά πτήσεις ή
χρήστες στο σύστημα. Το Backend φιλτράρει, ελέγχει τη μοναδικότητα (π.χ.
Username, Αριθμός Ταυτότητας) και εισάγει τα δεδομένα σειριακά,
επιτυγχάνοντας τον αυτοματισμό της διαμόρφωσης του συστήματος.
● Μηχανισμοί Ασφάλειας (Security Layer):
○ SQL Injection Protection: Όλα τα SQL ερωτήματα εκτελούνται
αποκλειστικά μέσω Prepared Statements (παραμετροποιημένων
δηλώσεων) με τη χρήση του PDO, εκμηδενίζοντας την πιθανότητα έγχυσης
κακόβουλου κώδικα.
○ XSS Protection & URL Validation: Ενσωματώθηκαν έλεγχοι με Regular
Expressions (Regex) στα URL για τον εντοπισμό και τον αποκλεισμό
«παράνομων» χαρακτήρων (π.χ. < > ' " ; \). Επιπλέον,
χρησιμοποιείται η συνάρτηση filter_input() για το Validation των
παραμέτρων και η htmlspecialchars() κατά την προβολή των
δεδομένων στο Frontend.
○ Password Hashing: Όλοι οι κωδικοί πρόσβασης αποθηκεύονται στη βάση
δεδομένων κρυπτογραφημένοι με τη χρήση του ισχυρού αλγορίθμου
password_hash().

Γ. Επίπεδο Δεδομένων (Database)
Το επίπεδο αυτό είναι υπεύθυνο για τη μόνιμη αποθήκευση και ανάκτηση των
πληροφοριών του συστήματος. Χρησιμοποιήθηκε το Σχεσιακό Σύστημα Διαχείρισης
Βάσεων Δεδομένων MySQL / MariaDB. Η επικοινωνία μεταξύ του Backend (PHP) και
της Βάσης έγινε αποκλειστικά μέσω της τεχνολογίας PDO (PHP Data Objects), κάνοντας
χρήση Prepared Statements για την πλήρη θωράκιση έναντι επιθέσεων τύπου SQL
Injection.
Σημείωση Αρχιτεκτονικής: Η ροή δεδομένων ακολουθεί αυστηρά τη διαδρομή:
Πελάτης (Browser) => Εξυπηρετητής (PHP) => Βάση Δεδομένων (MySQL). Κανένας
χρήστης δεν έχει απευθείας πρόσβαση στα δεδομένα χωρίς τη μεσολάβηση των ελέγχων
ασφαλείας του Backend.


2. Διαμόρφωσης, Διάταξης και Εκτέλεσης
Για την τοπική εκτέλεση και δοκιμή της εφαρμογής, χρησιμοποιήθηκε το
ολοκληρωμένο περιβάλλον ανάπτυξης XAMPP.
Βήματα Εγκατάστασης και Ρύθμισης:

1. Δημιουργία project/φακέλου στο κατάλογο htdocs του XAMPP.
(C:/xampp/htdocs/flight_app/).

a. Άνοιγμα του XAMPP Control Panel και Start τον διακομιστή Apache και τη βάση
δεδομένων MySQL.

b. Δημιουργία Βάσης Δεδομένων.

c. Εκτέλεση Εφαρμογής: διεύθυνση: http://localhost/flight_app/index.php.


3. Αντιμετώπιση και Υλοποίηση Απαιτήσεων της Εφαρμογής
Όλες οι λειτουργικές και μη λειτουργικές απαιτήσεις που τέθηκαν στην εκφώνηση
αναλύθηκαν και ενσωματώθηκαν πλήρως στον κώδικα. Συγκεκριμένα:

Α. Διαχείριση Ρόλων και Δικαιωμάτων Πρόσβασης (RBAC)
Υλοποιήθηκε μηχανισμός Session Authentication. Όταν ένας χρήστης δεν είναι
συνδεδεμένος, το σύστημα του αποδίδει αυτόματα τον ρόλο του Επισκέπτη (Visitor),
επιτρέποντάς του να δει μόνο την αρχική σελίδα, τη φόρμα σύνδεσης και εγγραφής.
Μετά τη σύνδεση, οι ρόλοι customer, flight_admin και system_admin ελέγχονται
αυστηρά στην κορυφή κάθε backend αρχείου με δομές ελέγχου, εμποδίζοντας μη
εξουσιοδοτημένη πρόσβαση μέσω URL (Privilege Escalation).

Β. Κανόνες Ακύρωσης Κράτησης & Αποδέσμευσης Θέσης
- Ακύρωση Κράτησης: Επιτρέπεται μόνο αν η πτήση δεν βρίσκεται σε μία από τις δύο
τελικές καταστάσεις (ΠΕΡΑΤΩΜΕΝΗ, ΑΚΥΡΩΜΕΝΗ). Η κράτηση δεν διαγράφεται
φυσικά από τη βάση (DELETE), αλλά το status της αλλάζει σε ΑΚΥΡΩΜΕΝΗ για
διατήρηση ιστορικού, ενώ οι θέσεις seat_row και seat_column μηδενίζονται (NULL).
- Αποδέσμευση Θέσης: Πελάτες με εισιτήριο NORMAL ή BUSINESS έχουν τη
δυνατότητα να αποδεσμεύσουν τη θέση τους (θέτοντας τα πεδία σε NULL) χωρίς να
ακυρώσουν την κράτηση, καθιστώντας τη θέση άμεσα διαθέσιμη. Η επιλογή αυτή
αποκλείεται αυστηρά για κατόχους ECONOMY εισιτηρίων.

Γ. Αλυσιδωτές Μεταβολές (Database Transactions & Cascades)
Στο αρχείο process_status_change.php, χρησιμοποιήθηκαν Transactions (PDO
Transactions) για τη διασφάλιση της ακεραιότητας των δεδομένων. Όταν μια πτήση
ακυρώνεται από τον Admin, ακυρώνονται αυτόματα όλες οι κρατήσεις των επιβατών
της. Αντίστοιχα, όταν μια πτήση περατώνεται, όλες οι ενεργές κρατήσεις μετατρέπονται
αυτόματα σε ΠΕΡΑΤΩΜΕΝΕΣ. Αν οποιοδήποτε ερώτημα αποτύχει, γίνεται rollback.

Δ. Ασφάλεια: SQL Injection, XSS και Έλεγχοι Εγκυρότητας
- SQL Injection: Αποτράπηκε πλήρως με τη χρήση Prepared Statements με
παραμετροποιημένες τιμές σε όλα τα SQL ερωτήματα.
- XSS (Cross-Site Scripting): Όλα τα δεδομένα που εισάγονται ή αποστέλλονται μέσω
φορμών φιλτράρονται. Στα URL ενσωματώθηκε έλεγχος με Regular Expressions (Regex)
Αναφορά Εργασίας, Σωτηρία Λάκη
για τον αποκλεισμό επικίνδυνων χαρακτήρων (π.χ. < > ' " ; \). Κατά την προβολή
δεδομένων, χρησιμοποιείται η συνάρτηση htmlspecialchars().
- Έλεγχοι Εγκυρότητας (Validation): Πραγματοποιείται έλεγχος ύπαρξης τιμών για όλα
τα υποχρεωτικά πεδία (π.χ. ΑΦΜ και Διεύθυνση κατά την εγγραφή) και έλεγχος τύπων
δεδομένων μέσω της filter_input() (π.χ. διασφάλιση ότι το ID ή η θέση είναι ακέραιοι
αριθμοί).


4. Ακαδημαϊκή και Πρακτική Εμπειρία

Η υλοποίηση του συγκεκριμένου έργου αποτέλεσε μια γέφυρα ανάμεσα στη θεωρία των
μαθημάτων και την πραγματική ανάπτυξη μιας web εφαρμογής. Καθώς ανέλαβα την εξ
ολοκλήρου ανάπτυξη του συστήματος ως ατομική εργασία, η διαδικασία αυτή με
ανάγκασε να αντιμετωπίσω στην πράξη όλα τα στάδια του κύκλου ζωής ενός
πληροφοριακού συστήματος, από τον σχεδιασμό των πινάκων στη βάση δεδομένων
μέχρι το τελικό UI.
Μέσα από αυτή την ενασχόληση, τα σημαντικότερα οφέλη που αποκόμισα είναι:
● Κατανόηση της Αρχιτεκτονικής στην Πράξη: Έγινε απόλυτα σαφές πώς
διαχωρίζεται το Frontend από το Backend. Το να βλέπεις πώς μια ενέργεια του
χρήστη στο browser μεταφράζεται σε κώδικα PHP και τελικά αλλάζει δεδομένα
στη MySQL, προσφέρει μια σφαιρική εικόνα που δεν αποτιμάται εύκολα μόνο με
τη θεωρία.
● Η Σημασία της Ασφάλειας από τον Σχεδιασμό: Η ανάγκη για προστασία της
εφαρμογής από SQL Injections και XSS επιθέσεις δεν αντιμετωπίστηκε ως μια
τυπική υποχρέωση, αλλά ως βασικό κομμάτι της συγγραφής του κώδικα. Η χρήση
των Prepared Statements και των φίλτρων εισόδου έγινε πλέον αυτονόητη
πρακτική.
● Διαχείριση Πολύπλοκης Λογικής: Προκλήσεις όπως οι αλυσιδωτές μεταβολές
στις κρατήσεις κατά την ακύρωση μιας πτήσης ή ο δυναμικός αποκλεισμός
θέσεων για Economy εισιτήρια, με βοήθησαν να αναπτύξω την αναλυτική μου
σκέψη και να καταλάβω πώς μετατρέπονται οι πραγματικοί επιχειρηματικοί
κανόνες σε δομημένο κώδικα.

Συμπερασματικά, το project αυτό μου έδωσε την αυτοπεποίθηση ότι μπορώ να σχεδιάσω
και να φέρω εις πέρας μια πλήρη, λειτουργική και ασφαλή δυναμική εφαρμογή,
κατανοώντας τις δυσκολίες αλλά και τις βέλτιστες πρακτικές του σύγχρονου web
development.
